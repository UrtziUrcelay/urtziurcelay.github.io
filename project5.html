Course Description

CS 350

This courses main objective is to learn the efficient representation and processing of complex 3D scenes in order to improve CPU & GPU throughput.Topics include spatial data structures (static: K-Dimensional (KD) Tree, Binary Space Partition-ing Tree, Dynamic: Hierarchical Bounding Volume (AABB) Tree), object-culling methods (occlusion and portal)and spatial intersection algorithms (Gilbert-Johnson-Keerthi(GJK) and Minkowski Portal Refinement (MPR)).The course culminates with the application and integration of these approaches into a unified CPU accelerated framework.

For this a custom 3D framework was developed from scratch.

External libreries used:

  • OpenGL API
  • ImGui (GUI)
  • Assimp (model loading only)

Most of the models used in the demos are copyrighted and are only used for demostration purposes.

Octree Spatial partitioning & Separating Axis Theorem

In the demo we can see how the Octree and the SAT work together to detect collisions in three dimensional objects.

Static objects can be placed in intermediate levels of the Octree if their triangles overlap two or more subdivisions of the lowest level, otherwise they are placed in the leaf nodes.
